Real-Time Systems: Space Invaders Game (my favorite game of all-time)

Written in C#; using Visual Studio and Depaul University’s Azul game engine.

The project took about 3 months

Design Patterns & Points of Emphasis

  • Object Pool, Proxy, Singleton, Template, Observer, Command, State, and Strategy.

  • It implements a collision system using a basic 2D grid system, which determines intersection between shape objects. These shape objects are held within an object, called a sprite (game object that is rendered to a display), that uses inheritance (from multiple abstract objects/interfaces) and managed in a double linked list.

My Favorite Parts

  • My implementation of the observers and the timer manager objects, which held multiple command objects. These commands would be added and processed when certain collisions, or time events elapsed. I created a sort that basically functions like an Insertion Sort, but the list is always in sorted order. It inserts an object into its correct position based on a delta time when it should be processed. *(I want to reiterate that I never called a sort on the list. For example; if I inserted a node arbitrarily and then called a sort on the entire list every time, that would be far less efficient)

  • Another concept I implemented into this project was a recycling of memory/objects, that are created when the program runs. My "graveyard" implementation was used/created for efficiency between levels. It was put together towards the end and had alot bugs. But, the primary concept pertains to a object that was hit or "destroyed" in the game, instead of letting it be flagged for garbage collection, I would put the object into a linked list where it was stored for the next round(hence the name graveyard). Alot of issues came along with this like resetting positions and determining what each object was inside the graveyard (Like is it a shield block or a specific alien, for example). But overall I got it to work with making little hacks in the game objects and adding information when they were stored.

Mobile Development: Rick&Morty Game

Written in Swift; using XCode and SpriteKit for iOS environment.

This took about 3 months

Concepts

  • It’s a basic 2D game that uses a grid system. The basic premise is to move the character across the screen to "catch" falling objects to score points before the time expires.

  • I used similar concepts from previous projects like timers. For example; I create events to "drop" objects from the top of the screen and also randomizes the velocity of those objects.

  • I also created some sprite objects, which were placed into an array, that serve as "stars" in the background. I loops through these objects and randomly "hide" or "reveal" the object. I did this to give it a sort of active background with flashing lights in space instead of a blank screen.

  • I used inheritance to create my own sprite objects that derived from the base SKSpriteNode object. This was primarily used for the double dispatching issue I had with the Contact Delegate, which ultimately determines what should happen when collisions occur.

  • Just to elaborate a little bit on the double dispatching issue, its essentially a way to figure out what two objects are coming into contact with one another by calling a certain function. The function derives from an interface or abstract object and depending on the type (Object-Oriented Principle) they have certain tasks they must do. Since all the objects derived from the base object, we need to create our own versions, like a wrapper around the base object, so we can perform more complicated or specific tasks.

Reasoning & Evolution

  • This project built on my previous experiences, but with a different environment and tools completely on my own.

  • My original idea was calling it the garbage collection game, which is sort of a play on words because of computer systems and it would be a game collecting trash in a park or something(because I hate when people liter). But, I'm a big fan of the Rick & Morty show and wanted to also make a really simple background by putting the game in space.

  • It was my first experience with macOS, as well, so to expand on my experience I used some built-in tools to edit and convert sound clips into .wav files. In turn, I used them in association with the collision system and timer mechanics of the game to prudce sound effects.
