Real-Time Systems: Space Invaders Game

Written in C#; using Visual Studio and Depaul University’s Azul game engine.

Design Patterns used: Object Pool, Proxys, Singletons, Templates, Observers, Commands, State, and Strategy. Implemented collision systems using a basic 2D grid system to determine intersection between shape objects These shape objects are held within an object, essentially sprites, (game objects that are rendered to a display) that uses inheritance (from a lot of abstract objects, like interfaces) and managed in a double linked list

Favorite portion: the observers and the timer manager objects, which held multiple command objects. These commands would be added and processed when certain collisions, or time events elapsed. I created a sort that basically functions like an Insertion Sort, but the list is always in sorted order. It inserts an object into its correct position based on a delta time when it should be processed. *(I want to reiterate that I never called a sort on the list. For example; if I inserted a node arbitrarily and then called a sort on the entire list every time, that would be far less efficient)

Mobile Development: Rick&Morty Game

Written in Swift; using XCode and SpriteKit for iOS environment.

It’s a basic 2D game that uses a grid system. The basic premise is to move the character across the screen to “catch falling objects” to score points before the time expires. This project built on my previous experience, but with a different environment and tools completely on my own. I used inheritance to create my own sprite objects that derived from the base SKSpriteNode object. This was primarily used for the double dispatching issue I had with the Contact Delegate, which ultimately determines what should happen when collisions occur.
